【面向玩家】快速上手

1.1 游戏介绍

《地球Online：现实演化版》是一款无强制主线、高自由度的现实世界沙盒模拟游戏，你将以世界观测者+规则设定者的身份，见证从地球初生、生命萌芽到人类文明崛起的全流程，也可以自定义规则，打造专属的地球演化路线。

核心特色：

• 无等级、无强制任务，世界完全由时间推进与规则设定驱动

• 全球11个国家分服同步运行，独立演化，支持随时切换

• 完整还原四季特性、节日民俗、随机自然灾害、文明阶段迭代

• 国服专属人生模拟主线，从出生到终老，还原真实人生轨迹

• 支持玩家自制MOD，修改世界规则、添加自定义事件、扩展玩法

1.2 安装与运行

1.2.1 预编译包运行（推荐普通玩家）

1. 前往项目 GitHub Release 页面，下载对应平台的最新预编译包：

◦ Windows：EarthOnline_Windows.zip

◦ Mac OS：EarthOnline_MacOS.zip

2. 解压压缩包到任意非中文路径

3. 双击运行程序：

◦ Windows：双击 EarthOnline.exe

◦ Mac OS：打开终端，进入解压目录，执行 chmod +x EarthOnline && ./EarthOnline

4. 程序启动后会自动创建所需的目录与默认配置文件，无需额外配置。

1.2.2 注意事项

• Windows 平台请勿放在中文路径下，避免存档/MOD加载异常

• Mac OS 若提示「无法打开」，请前往「系统设置→隐私与安全性」，允许程序运行

• 游戏全程仅需键盘操作，按照控制台提示输入数字/文本即可

1.3 核心玩法规则

1.3.1 时间系统

游戏时间完全贴合现实逻辑，是世界演化的核心驱动：

• 最小时间单位：季节（春/夏/秋/冬），四季轮换对应世界基础变化

• 基础时间单位：年，4个季节=1个完整自然年，是规则修改、世界迭代的核心周期

• 进阶时间单位：世纪，100年=1个世纪，用于划分宏观地质、文明大阶段

• 单局人生上限100年，年龄随年份推进自动增长，100年时触发人生结局

1.3.2 纪元与文明演化

游戏按照地球真实演化逻辑，划分三大核心纪元，无干预状态下会自动完成全流程迭代：

• AR纪元（太古代/元古代）：地球初生，仅有原始微生物、藻类，大气无氧，地表荒芜

• PT纪元（古生代/中生代）：生物登陆，脊椎动物崛起，恐龙成为地球主宰

• PH纪元（新生代）：恐龙灭绝，哺乳动物繁衍，灵长类进化，人类文明诞生与发展

• 人类文明分为原始社会、奴隶社会、封建社会、近代社会、现代社会五大阶段，会随科技、经济发展自动进阶

1.3.3 全球分服系统

游戏覆盖全球11个核心国家分服，所有国家共用一套时间轴，同步演化：

• 可选分服：中国（国服）、美国（美服）、日本、泰国、俄罗斯、法国、德国、意大利、澳大利亚、新加坡、马来西亚

• 开局可选定所属分服，全程观测/调控对应国家的演化，也可随时切换分服查看他国状态

• 各国独立运行规则、互不干扰，可单独修改单一国家设定，也可同步修改全球规则

1.3.4 四季与自然灾害

• 四季专属特性：每个季节有固定的环境、社会影响，南北半球特性自动适配

◦ 春季：气温回升，农耕播种，生态提升，户外活动增多，流感、过敏类疾病高发

◦ 夏季：全年最高温，植被繁茂，水上活动集中，极端高温影响劳作，台风、洪涝高频

◦ 秋季：气温下降，农作物丰收，经济、民生提升，婚嫁、搬迁、商务活动旺季

◦ 冬季：全年最低温，农耕休耕，室内活动增多，年末节庆集中，暴雪、寒潮灾害高频

• 自然灾害：分为气象灾害（绑定季节触发）和地质灾害（全域随机触发），会直接影响生态、民生、经济，可通过科技发展、生态保护降低损失

1.3.5 节日与任务系统

• 节日体系：分为全球通用节日和各国本土专属节日，固定时间触发，解锁专属buff和支线任务

◦ 全球通用：元旦、劳动节、儿童节、国庆节等

◦ 国服专属：春节、元宵节、清明节、端午节、中秋节、重阳节、七夕节

◦ 其余分服均有对应本土传统节日，自动匹配触发

• 任务系统：仅国服玩家触发专属人生任务链，其余分服无强制任务

◦ 主线任务：贴合人生阶段自动触发，分为「学会基本技能→系统学习知识→步入职场工作→组建家庭结婚→孕育养育子女」五大必经阶段，无失败惩罚

◦ 支线任务：根据出生点省份/地市专属生成，包含传统民俗、地域特色活动，完成可解锁成就、社交好感度

1.3.6 规则设定调控

• 每年冬季年末结算节点，开放设定修改权限，可一键修改全局/单一国家的核心规则，实现文明跨越

• 可添加长期趋势设定，指定【起始年份-截止年份】内，某一核心指标（科技、经济、生态等）按固定趋势自动变化

• 所有设定修改均会实时生效，同步影响世界演化走向

1.4 MOD安装与使用

游戏支持玩家自制/第三方MOD扩展玩法，MOD为纯TXT文本格式，无需编程基础即可使用。

安装步骤

1. 下载/自制MOD文件，格式为 .txt

2. 打开游戏根目录下的 mod 文件夹，可新建子目录分类存放MOD

3. 将MOD文件放入 mod 文件夹（或子目录）

4. 重启游戏，程序会自动扫描并加载所有有效MOD，启动页会显示已加载的MOD列表

注意事项

• MOD仅支持 .txt 格式，请勿放入其他类型文件

• 冲突MOD会以后加载的为准，建议同类型MOD仅保留一个

• 若MOD加载异常，可查看游戏根目录 log/game_run.log 文件排查问题

1.5 存档管理

游戏支持随时存档、读档，无视原设定的不可逆规则，存档为纯TXT格式，支持手动修改。

存档操作

• 保存存档：游戏主菜单选择「保存游戏」，输入存档名称，程序会自动在 saves 文件夹生成对应存档文件

• 加载存档：游戏开始菜单选择「加载存档」，或主菜单选择「加载存档」，输入存档名称即可加载

• 存档备份：可直接复制 saves 文件夹内的TXT文件进行备份，分享、转移只需复制对应TXT文件

存档规则

• 存档文件不可修改后缀名，否则无法识别

• 存档包含当前世界所有状态、玩家信息、MOD自定义变量，加载后完全还原

• 不同版本的存档可能存在兼容性问题，建议大版本更新前备份存档

1.6 玩家常见问题

1. 游戏启动后中文乱码怎么办？

◦ Windows平台：确保程序放在非中文路径，若仍乱码，可右键CMD窗口→属性→选项，勾选「使用Unicode UTF-8」

◦ Mac OS：确保终端编码为UTF-8，可在终端设置→描述文件→高级中，将文本编码改为「UTF-8」

2. MOD放进去不加载怎么办？

◦ 检查MOD文件后缀是否为 .txt，文件是否损坏

◦ 查看 log/game_run.log 日志，确认MOD语法是否有误

◦ 确保MOD放在 mod 文件夹内，而非根目录

3. 存档加载失败怎么办？

◦ 确认存档名称输入正确，无需加 .txt 后缀

◦ 确认 saves 文件夹内存在对应存档文件

◦ 检查存档文件是否被修改过，语法是否符合规范

4. Mac OS提示无法打开程序怎么办？

◦ 打开「系统设置→隐私与安全性」，下滑找到「已阻止使用」，点击「仍要打开」

◦ 若仍无法打开，打开终端，进入程序目录，执行 xattr -d com.apple.quarantine EarthOnline

2.1 项目仓库与整体架构

2.1.1 项目信息

• 项目仓库：https://github.com/你的用户名/地球Online（替换为实际仓库地址）

• 开发语言：标准C++（C++11及以上），无第三方依赖，无平台专属API

• 核心特性：跨平台原生编译、轻量MOD脚本引擎、可扩展剧本系统、全序列化存档

2.2 开发环境配置与编译

2.2.1 通用要求

• 编译器支持C++11及以上标准

• 无第三方库依赖，仅使用C++标准库

• 代码采用UTF-8编码，避免中文乱码

2.2.2 Windows环境配置

推荐使用MinGW-w64编译器，提供两种可靠安装方案：

方案1：WinLibs免安装包（推荐）

1. 前往 WinLibs官网，下载最新版本的免安装包（推荐UCRT版本，64位）

2. 解压压缩包到任意非中文路径，将解压后的 mingw64/bin 文件夹添加到系统环境变量PATH

3. 打开CMD，执行 g++ --version，输出版本号即为配置成功

方案2：SourceForge官方安装包

1. 前往 MinGW-w64 SourceForge页面，下载安装包

2. 安装时，架构选择 x86_64，线程模型选择 win32，异常模型选择 seh

3. 安装完成后，将安装目录下的 mingw64/bin 添加到系统环境变量PATH

4. 打开CMD，执行 g++ --version 验证配置

编译命令
# 进入代码根目录
cd D:\地球Online
# 编译生成可执行文件
g++ -o EarthOnline main.cpp -std=c++11
# 运行程序
EarthOnline.exe
2.2.3 Mac OS环境配置

1. 安装Xcode命令行工具，打开终端执行：
xcode-select --install
2. 验证安装：执行 g++ --version，输出版本号即为成功

3. 编译命令：
# 进入代码根目录
cd ~/Desktop/地球Online
# 编译生成可执行文件
g++ -o EarthOnline main.cpp -std=c++11
# 赋予执行权限并运行
chmod +x EarthOnline && ./EarthOnline
2.2.4 Xcode项目配置

1. 打开Xcode，新建「Command Line Tool」项目，语言选择C++

2. 将项目仓库内的所有代码文件导入项目

3. 选中项目→「Build Settings」→「C++ Language Dialect」，选择 C++11 及以上版本

4. 按下 Cmd+R 即可编译运行

2.4.3 剧本编写规则

• 一行一个配置项，格式严格为 键=值，无多余空格

• # 开头的行视为注释，程序自动忽略

• 不支持行内注释，注释需单独占一行

• 重复字段以最后一次出现的为准

• 缺失字段会使用程序内置默认值，不会导致启动失败

2.5 MOD开发全语法规范

MOD为纯TXT文本格式，内置轻量脚本引擎，支持变量定义、条件判断、嵌套逻辑、玩家交互、键值修改，无需编译，放入mod目录即可生效。

2.5.1 MOD基础信息（必填）

所有MOD必须在文件开头填写基础信息，否则无法被正确识别：
# MOD基础信息（必填）
MOD_NAME=MOD名称
MOD_AUTHOR=作者名
MOD_VERSION=版本号
MOD_DESC=MOD功能描述
# MOD执行时机（可选，默认启动时执行）
#EXEC start/season/newyear
• #EXEC：指定MOD执行时机，可选值：

◦ start：游戏启动时执行一次

◦ season：每个季节切换时执行

◦ newyear：每年新年伊始时执行

2.5.2 注释规则

• 单行注释：以 # 开头，整行被程序忽略

• 行内注释：语句末尾添加 # 注释内容，不影响语句执行
# 这是单行注释
VAR AGE=18 # 这是行内注释
2.5.3 自定义变量定义

• 语法：VAR 变量名=值

• 支持数字型、字符串型变量，变量名不区分大小写

• 变量全局生效，可跨语句、跨IF块调用，支持跨存档持久化保存
# 定义数字变量
VAR PLAYER_LEVEL=1
VAR TECH_MULTIPLE=3.0
# 定义字符串变量
VAR MOD_TITLE=超级扩展模组
2.5.4 键值修改语句

• 语法：SET 键名=值

• 值可以是字面量、自定义变量、算术表达式

• 可修改全局设定、国家属性、自定义变量
# 直接修改字面量
SET BASE_TECH_SPEED=3.0
# 修改变量值
SET BASE_TECH_SPEED=USER_TECH_MULT
# 表达式修改
SET BASE_TECH_SPEED=TECH_MULTIPLE * 2
2.5.5 输入输出语句

• 输出语句：PRINT 输出内容，支持变量占位符 ${变量名}

• 输入语句：INPUT 提示文本 变量名，接收玩家输入的数字，存入指定变量
# 输出语句，支持变量替换
PRINT 当前年份是${YEAR}，你的年龄是${PLAYER_AGE}
# 输入语句，接收玩家输入
INPUT 请输入科技倍率 USER_TECH_MULT
2.5.6 IF条件判断语句

支持无限层嵌套、与或非逻辑、大小比较，条件中可使用内置变量、自定义变量。

基础语法
IF 条件表达式 THEN
    # 条件成立时执行的语句
ELSE
    # 条件不成立时执行的语句（可选）
END IF
运算符支持
运算符类型 支持符号 说明 
比较运算符 > < >= <= == != 大于/小于/大于等于/小于等于/等于/不等于 
逻辑运算符 AND OR NOT 与/或/非（不区分大小写） 
算术运算符 + - * / 加减乘除 

条件表达式示例
# 单条件判断
IF YEAR >= 2030 THEN
    PRINT 2030年已到，科技速度翻倍！
    SET BASE_TECH_SPEED=2.0
END IF

# 多条件与逻辑
IF PLAYER_AGE >= 18 AND COUNTRY_TECH >= 2.0 THEN
    PRINT 你已成年且科技达标，解锁超级加成！
    SET BASE_TECH_SPEED=3.0
END IF

# 非逻辑
IF NOT SEASON == 3 THEN
    PRINT 当前不是冬季，无法修改设定！
END IF

IF PLAYER_AGE >= 18 THEN
    PRINT 你已成年，解锁成长路线选择！
    IF GROWTH_TYPE == 0 THEN
        INPUT 请选择成长路线（1=科技 2=经济） GROWTH_TYPE
        IF GROWTH_TYPE == 1 THEN
            SET COUNTRY_TECH=1.5
        ELSE

    END IF
END IF

2.6.2 读写规范

• 分段标识以 [段名] 格式单独占一行，段内为 键=值 格式的配置项

• 程序读取时按段解析，缺失段不会导致加载失败，会使用默认值填充

• 手动修改时，需保证键名正确，值的类型与字段匹配，否则会使用默认值

• 存档加载时，会自动还原MOD自定义变量，保证MOD逻辑连续

3.2 常见问题排查

3.2.1 编译相关

1. Windows提示 g++ 不是内部或外部命令

◦ 检查MinGW-w64的 bin 目录是否已添加到系统环境变量PATH

◦ 重启CMD/终端，重新执行命令

◦ 确保环境变量路径无中文、无空格

2. Xcode编译报错 No matching member function for call to push_back

◦ 检查结构体初始化列表的参数顺序，是否与结构体成员声明顺序一致

◦ 确保C++ Language Dialect设置为C++11及以上版本

◦ 检查初始化列表的参数类型是否与结构体成员类型匹配

3. 编译报错 dirent.h: No such file or directory

◦ Windows平台请使用MinGW-w64编译器，自带dirent.h库

◦ Visual Studio需手动安装dirent.h第三方库，推荐使用MinGW-w64避免该问题

3.2.2 运行相关

1. MOD脚本执行异常

◦ 检查IF语句是否成对使用 IF 和 END IF，嵌套层级是否正确

◦ 检查条件表达式的运算符是否正确，AND/OR 前后必须有空格

◦ 查看 log/game_run.log 日志，定位具体执行错误的行

2. 存档加载后MOD变量丢失

◦ 确保变量名一致，变量名不区分大小写

◦ 检查 user_var/mod_var_save.txt 文件是否存在、是否被修改

◦ 确保MOD在存档加载前已被正确加载
